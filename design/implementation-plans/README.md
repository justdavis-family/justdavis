# Implementation Plans

This directory contains engineering and technical implementation plans for projects in this monorepo.

## Purpose

Engineering implementation plans capture technical implementation details
  and actionable tasks for building the system, including:

- Detailed implementation steps for each component or feature.
- Specific code changes and file modifications.
- Configuration and deployment instructions.
- Testing and validation procedures.
- Rollout and monitoring plans.

These documents focus on the very specific **what** and **how**
  of a feature or other implementation task.
They are often based on [product requirements](../product-requirements/)
  and [engineering design documents](../engineering-designs/).

Most often, implementation plans are created to guide LLM-based development agents
  through the implementation process.
When created by/for human developers,
  they more often serve as a brainstorming and planning tool,
  rather than a strict step-by-step guide.
